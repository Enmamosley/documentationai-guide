## Por qué importa

El código de WCG App está plagado de siglas (R20, F200, EMU, CFDI, SIATEL, FTTH). Sin este glosario el modelo de datos parece arbitrario; con él, cada tabla cuadra con un documento real del cliente.

<Callout kind="info" collapsed="false">
  WCG no es un cliente final — es **contratista de Telmex / CARSO**. Casi todos los términos vienen del lado Telmex.
</Callout>

## Entidades de dominio

<Columns cols={2}>
  <Card title="OS — Orden de Servicio" icon="clipboard-list">
    Trabajo solicitado por Telmex (alta de cliente, migración, reparación). Tiene **folio WCG** + **folio SIATEL** (Telmex). Estado: Capturada → Liquidada → Facturada → Pagada.
  </Card>
  <Card title="ONT" icon="router">
    Equipo terminal (modem de fibra) que se instala en casa del cliente. Identificado por **alfanumérico**. Ciclo: En almacén → Con técnico → Liquidada → Facturada.
  </Card>
  <Card title="SKU" icon="hash">
    Clave AXAPTA del material que CARSO surte. Cada SKU tiene 3 factores Telmex (`alta_ftth`, `mig_ftth`, `mig_ftth_vieja`) que dicen cuántas piezas consume cada tipo de OS.
  </Card>
  <Card title="Remisión" icon="truck">
    Documento con el que **CARSO envía material a WCG**. Estados: Recibida → Confirmada (default `Recibida` al registrar). Al confirmar, el stock entra al almacén destino atómicamente.
  </Card>
  <Card title="Factura emitida (CFDI)" icon="file-text">
    Factura que **WCG emite a CARSO**. Va en lotes con CARSO y pasa por: Borrador → Emitida → En revisión → Aceptada → Por pagar → Pagada.
  </Card>
  <Card title="EMU" icon="calendar">
    **E**stimación **M**ensual de **U**nidades. Cobranza semanal por técnico × concepto. Cuando una factura asociada se marca Pagada, las filas EMU pasan a Cobrado por trigger SQL.
  </Card>
</Columns>

## Tipos de servicio

| Tipo | Factor que aplica | Descripción |
|---|---|---|
| **Alta FTTH** | `factor_alta_ftth` | Cliente nuevo con fibra |
| **Migración FTTH** | `factor_mig_ftth` | Cliente cobre → fibra (terminal nuevo) |
| **Migración FTTH vieja** | `factor_mig_ftth_vieja` | Cliente cobre → fibra (terminal existente) |

El factor multiplica las **altas** de la OS para sugerir cuánto material se necesita. Por ejemplo, un SKU con factor 1.0 en Alta FTTH y altas=3 sugiere 3 piezas.

## Estados (al detalle)

### OS

```
Capturada  →  Liquidada  →  Facturada  →  Pagada
     ↓
  Cancelada
```

- **Capturada**: el técnico (o operaciones) llenó el R20 desde campo.
- **Liquidada**: el material se consumió del kit del técnico (auto-consumo desde stock por mochila) y la OS está lista para reportar.
- **Facturada**: incluida en una factura CFDI emitida a CARSO.
- **Pagada**: CARSO depositó.

### ONT

```
En almacén  →  Con técnico  →  Liquidada  →  Facturada
                     ↓
                  Dañada
```

### Factura emitida

```
Borrador → Emitida → Enviada al lote → En revisión CARSO →
  Aceptada → Por pagar → Pagada parcial → Pagada
```

Rechazada y Cancelada salen del flujo en cualquier punto.
