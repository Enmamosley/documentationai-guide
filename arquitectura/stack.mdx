## Convenciones

<Steps>
  <Step title="Server + Client split" icon="layers" title-type="p">
    Cada pantalla interactiva está partida en:
    - `page.tsx` (Server Component) — hace `fetch` a Supabase con cookies vía `getSupabaseServer()`.
    - `<Pantalla>Client.tsx` (Client Component) — recibe data como props, maneja UI.

    El servidor cachea con `force-dynamic` por default; mover a `force-static + ISR` cuando se quiera.
  </Step>

  <Step title="Sin Tailwind" icon="palette" title-type="p">
    El diseño viene del bundle Claude Design en `prototype/`. Se mantuvo CSS tradicional con variables como design tokens (`globals.css`) para pixel-perfect.
  </Step>

  <Step title="Mutaciones via Server Actions" icon="zap" title-type="p">
    Nunca se mutan datos desde Client Components con fetch. Todas las mutaciones pasan por `actions/*.ts` que usan `getSupabaseServer()` (JWT del usuario llega a Postgres → RLS evalúa).
  </Step>

  <Step title="Sólo desktop 1280–1440px" icon="monitor" title-type="p">
    Regla `min-width: 1280px` en `.app` fuerza scroll horizontal antes que colapsar el layout. Móvil queda fuera de scope hasta Fase 2.
  </Step>
</Steps>

npm run build    # producción
npm run lint     # eslint
