## RLS (Row-Level Security)

<Callout kind="alert" collapsed="false">
  **RLS está ACTIVO en todas las tablas.** Sin sesión válida, todas las queries fallan con `42501`. No se confía en el cliente — la seguridad vive en Postgres.
</Callout>

### Lectura

Todas las tablas tienen una policy uniforme:

```sql
CREATE POLICY <tabla>_select_all ON <tabla>
  FOR SELECT TO authenticated USING (true);
```

Cualquier usuario autenticado puede leer todo. Sin sesión → no ve nada.

### Mutaciones

Cada tabla tiene policies por rol via `current_user_role()`:

| Tabla | INSERT | UPDATE |
|---|---|---|
| `os` | admin · operaciones · tecnico (suya) | admin · operaciones · tecnico (suya) |
| `onts` | admin · almacenista | admin · operaciones · almacenista · tecnico (asignada) |
| `facturas` | admin · contable | admin · contable |
| `inventory` | admin · almacenista | admin · almacenista (vía triggers atómicos) |
| `remisiones` | admin · almacenista · operaciones | admin · almacenista |
| `emu_rows` | trigger SQL (auto al liquidar OS) | admin · contable |
| `user_profiles` | admin | admin |

### Helper SQL

```sql
CREATE FUNCTION current_user_role() RETURNS user_role AS $$
  SELECT rol FROM user_profiles WHERE id = auth.uid()
$$ LANGUAGE sql STABLE;
```

## Triggers atómicos

<Steps>
  <Step title="liquidar_os → consume mochila" icon="package-minus" title-type="p">
    Cuando `os.estado` pasa a `Liquidada`, un trigger BEFORE UPDATE descuenta de `inventory` (mochila del técnico) las cantidades de `os_partidas`. Si no hay stock, falla con `23514` y la mutación se aborta.
  </Step>

  <Step title="confirmar_remision → incrementa almacén" icon="package-plus" title-type="p">
    Al pasar `remisiones.estado` a `Confirmada`, las partidas suben el `disponible` del almacén destino. Atómico via row-level locking.
  </Step>

  <Step title="factura_pagada → EMU cobrado" icon="check-circle" title-type="p">
    Cuando `facturas.estado` pasa a `Pagada`, todas las `emu_rows` con `factura_id = NEW.id` pasan a `Cobrado` y se sella `fecha_cobro`.
  </Step>

  <Step title="prevent_role_escalation" icon="shield" title-type="p">
    Trigger en `user_profiles` BEFORE UPDATE: si `NEW.rol != OLD.rol` y `auth.uid() != admin`, se aborta. (Bug 4 cerrado en `6436986`.)
  </Step>
</Steps>
