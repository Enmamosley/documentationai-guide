## Qué muestra

`/programa-obra` es el **Gantt semanal** que CARSO usa para planificar carga de trabajo por zona y técnico. Cada barra es un compromiso de instalaciones / migraciones por semana ISO.

## Vista

- **Filas**: técnicos agrupados por zona (Norte, Sur, etc.)
- **Columnas**: semanas (ISO) — usualmente 6-8 semanas hacia adelante
- **Barras**: compromiso de altas o migraciones para esa semana × técnico
- **Indicadores**: % avance vs comprometido, sobre/sub-asignación

## Permisos

| Acción | Roles |
|---|---|
| Ver listado | admin, **operaciones** |
| Editar compromisos | — *pendiente* |

<Callout kind="alert">
  **Estado actual: read-only.** La UI muestra los programas existentes (con su PEP, supervisor, % avance, etc.) pero **no hay editor visual del Gantt**. El botón "Nuevo programa" fue removido (era cosmético). Si necesitás cargar un programa nuevo, hacelo manualmente en DB sobre `programas_obra` + `programa_dias`. El empty state lo indica explícitamente.
</Callout>

<Callout kind="info">
  Este módulo es de planeación. No genera OS automáticamente — sólo es el plan. Las OS reales se capturan desde `/os/nueva` cuando el técnico ya hizo el servicio.
</Callout>
