## Qué guarda

Por cada técnico, el expediente cumple los requisitos de **REPSE / IMSS**:

- Datos personales: nombre, alias, teléfono, zona
- PII fiscal: RFC, CURP, IMSS (NSS), INE
- Activos asignados: herramienta + vehículo
- Documentos adjuntos: identificación, contrato, etc.

<Callout kind="alert">
  El expediente es **PII sensible**. Sólo **admin** puede editarlo. El propio técnico ve su perfil pero no puede modificar campos fiscales.
</Callout>

## Foto del técnico

Sub-panel en Expediente muestra la foto (96px preview en panel, 36px circular en header).

- **Storage**: bucket privado `wcg-docs` con prefijo `tecnico-fotos/<id>.<ext>`
- **Acceso**: signed URL con TTL 10 min (generada server-side en `page.tsx`)
- **Subir / Reemplazar**: admin u operaciones, `image/*` hasta 5 MB (upsert)
- **Eliminar**: sólo admin (NULL el campo + `remove()` el blob)

```
src/lib/actions/tecnicos.ts
  subirFotoTecnicoAction(tecnicoId, formData)      # admin · operaciones
  eliminarFotoTecnicoAction(tecnicoId)             # admin
```

## Tab Inventario actual

Vista en vivo de lo que el técnico tiene **en este momento** en su mochila. Útil para detectar:

- Si necesita más cable / fibra (alerta de surtido).
- **Descuadres** (items en negativo por consumo sin surtido previo) — banner warning rojo.
- Material olvidado al cambiar de proyecto.

4 stat cards: SKUs distintos (split positivos/cero/negativos), unidades totales, valor estimado MXN, última fecha de movimiento. Tabla con filas en rojo si `disponible < 0`, gris si `= 0`.

```
src/lib/db.ts
  fetchInventarioDeTecnico(tecnicoId): Promise<{ items: StockTecnicoItem[] } | null>
```

Empty state lleva a `/almacen/surtir`.

## Tab Documentos

Lista de documentos vinculados al técnico (PII, contratos, fotos). Botón "Subir otro" lleva a `/sistema/documentos?tecnico_id=<id>` con el form pre-fillado.

```
src/lib/db.ts
  fetchDocumentosDeTecnico(tecnicoId): Promise<Documento[]>
```

RLS: el propio técnico ve sólo los suyos; admin/operaciones/contable ven todos los del técnico consultado.

## Permisos

| Acción | Roles |
|---|---|
| Ver listado y detalle | todos excepto **técnico** |
| Editar expediente (PII) | **admin** |
| Gestionar herramienta + vehículo | **admin** |
| Imprimir credencial / ficha | admin |

## Server Actions

```
src/lib/actions/tecnicos.ts
  editarExpedienteAction(tecnicoId, patch)
    // Admin-only. Upsert sobre tecnicos_expediente (crea fila si no existe).

  agregarHerramientaAction(tecnicoId, input)
  retirarHerramientaAction(herramientaId, tecnicoId)
  agregarVehiculoAction(tecnicoId, input)
  retirarVehiculoAction(vehiculoId, tecnicoId)
    // No es bulk — cada item se agrega/retira individualmente. La fila
    // queda marcada con `vigente=false` al retirar (historial preservado).

  subirFotoTecnicoAction(tecnicoId, formData)       // admin · operaciones
  eliminarFotoTecnicoAction(tecnicoId)              // admin
```
