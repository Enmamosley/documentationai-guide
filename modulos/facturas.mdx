## Qué muestra

`/facturas` lista todos los **CFDIs emitidos por WCG a CARSO** con tabs por estado. Cada fila es un comprobante fiscal con serie + folio, RFC receptor (siempre CARSO), monto, UUID SAT y lote CARSO.

<Callout kind="alert">
  No confundir con "facturas recibidas". WCG **no** compra material externo — CARSO surte vía remisión. Las facturas que importan acá son las que WCG **emite** a CARSO para cobrar.
</Callout>

<Callout kind="info">
  La tabla en BD es **`facturas_emitidas`** (enum `factura_emitida_estado`). La tabla legacy `facturas` (con enum `factura_estado`) fue eliminada en el commit `60248f3` (Bug 13). Ya sólo existe una tabla de facturas en el sistema.
</Callout>

## Ciclo de vida (10 estados)

```
Borrador → Emitida → Enviada al lote → En revisión CARSO →
  Aceptada → Por pagar → Pagada parcial → Pagada
                    ↓
                Rechazada
                Cancelada
```

| Estado | Quién la mueve |
|---|---|
| **Borrador** | Contabilidad arma el CFDI |
| **Emitida** | Se selló con el PAC; tiene UUID SAT |
| **Enviada al lote** | Se envió a CARSO dentro de un lote semanal |
| **En revisión CARSO** | CARSO la está validando |
| **Aceptada** | CARSO la validó; entra al ciclo de pago |
| **Por pagar** | Aceptada y a la espera de depósito |
| **Pagada parcial** | CARSO pagó una parte |
| **Pagada** | Pagada total → trigger marca filas EMU como Cobrado |
| **Rechazada** | CARSO la rechazó; requiere reproceso |
| **Cancelada** | Se canceló ante SAT |

## Detalle `/facturas/[id]`

- **Datos fiscales**: serie/folio, UUID SAT, RFC emisor (WCG) y receptor (CARSO), subtotal/IVA/total.
- **Conceptos**: lista de OS / EMU que conforman la factura.
- **XML + PDF adjuntos**: parser CFDI 4.0 extrae los datos del XML al subirlo (`src/lib/cfdi-parser.ts`).
- **Lote CARSO**: número de lote para tracking.
- **Historial**: cada cambio de estado con timestamp y usuario.

<Steps>
  <Step title="Subir XML" icon="upload" title-type="p">

  </Step>

## Crear factura nueva (`/facturas/nueva`)

UI para que el contable arme una factura asociando EMUs pendientes
(commit `3bd7c67`, B3):

<Steps>
  <Step title="Datos generales" icon="file-plus" title-type="p">
    Número (ej. `A-0001`), contratista, fecha, proyecto, lote envío
    CARSO opcional + códigos contables opcionales (obra CIC, orden de
    compra).
  </Step>
  <Step title="Seleccionar EMUs pendientes" icon="check-square" title-type="p">
    Tabla con todas las filas EMU en `estado_cobro = 'pendiente'`.
    Checkboxes por fila o "Seleccionar todas". 4 stat cards en vivo
    muestran el conteo y los totales (subtotal, IVA, total).

    La **tasa de IVA** viene de `empresa_config.tasa_iva` (default 0.16).
    Régimen frontera norte usa 0.08, régimen exento usa 0. El label del
    stat card se ajusta dinámico — "IVA (16%)" / "IVA (8%)" / "IVA (0%)".
    Editable en `/sistema/empresa`. El campo "Contratista" pre-rellena
    con `empresa_config.razon_social`.
  </Step>
  <Step title="Crear" icon="zap" title-type="p">
    INSERT en `facturas_emitidas` con estado `Borrador` + `uuid`
    placeholder. UPDATE bulk en las EMUs seleccionadas para asociarlas
    (`factura_emitida_id` + `estado_cobro = 'facturado'`). Rollback
    automático: si la asociación falla, DELETE la factura recién creada.
  </Step>
</Steps>

## Server Actions

```
src/lib/actions/facturas.ts
  actualizarCobranzaAction(facturaId, patch)
    // Mueve la factura entre estados (Borrador → … → Pagada),
    // setea fechas (envio_carso, aceptacion_carso, pago), lote_envio,
    // retencion_isr/iva, notas_cobranza. Patch parcial.
    // ⚠️ El trigger emu_avanzar_cobrado SÓLO dispara cuando estado pasa
    // a 'Pagada' completa, no con 'Pagada parcial' (Bug 6).

  subirCFDIAction(facturaId, formData)
    // FormData con xml + opcional pdf. Sube a Storage privado
    // cfdi-emitidos, parsea XML con fast-xml-parser (cfdi-parser.ts).
    // Si el XML trae un UUID distinto, gana el XML (es el registro fiscal).

  getCFDIDownloadUrlAction(facturaId, type: 'xml' | 'pdf')
    // Signed URL (5 min) para descargar el adjunto.

  crearFacturaEmitidaAction(input: CrearFacturaInput)
    // B3: INSERT factura en Borrador con uuid placeholder + asociar
    // EMUs seleccionadas (estado_cobro → 'facturado'). Rollback automático.
```

<Callout kind="info">
  El uuid arranca como `pendiente-<timestamp>` y se reemplaza al subir
  el XML real con `subirCFDIAction`. Esto permite armar la factura
  internamente antes de timbrarla en el PAC.
</Callout>
