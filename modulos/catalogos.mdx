## Qué contiene

`/catalogos` es el hub de todos los maestros. **8 catálogos editables** + **2 enums hardcoded** del contrato Telmex.

| Slug | Editable | Notas |
|---|---|---|
| `skus` | ✅ | Material AXAPTA con SIATEL + 3 factores Telmex |
| `tecnicos` | ✅ | Padrón con alias/zona, soft-delete vía `activo` |
| `almacenes` | ✅ | Sedes físicas |
| `copes` | ✅ | Centros operativos Telmex (CTMAC, CTVLL…) — expediente técnico + EMU |
| `empresas-empleadoras` | ✅ | Empresas REPSE bajo las que están los técnicos |
| `tipos-tarea` | ✅ | Conceptos A4/CO/FO/QI de la EMU semanal |
| `unidades-medida` | ✅ | Catálogo SAT — usado en SKUs y CFDIs |
| `clientes` | ✅ | Razones sociales receptoras de CFDI (típicamente CARSO) |
| `tipos-servicio` | 🔒 enum | Hardcoded — contrato Telmex (Alta nuevo, Migración…) |
| `estados-os` | 🔒 enum | Hardcoded — ciclo de vida (Capturada/Liquidada/…) |

<Callout kind="info">
  **Día 0**: TECNICOS y ALMACENES vivían hardcoded en TS hasta el commit `78fb761`. Ahora son DB-only — el admin los gestiona desde la UI sin tocar código. El `CatalogosBaseProvider` en el layout `(app)` fetcha activos una vez y los expone a los client components vía `useCatalogosBase()`.
</Callout>

## CRUD inline

Cada catálogo editable comparte el patrón:

<Steps>
  <Step title="Toolbar con search + count + 'Nuevo'" icon="search" title-type="p">
    Búsqueda por PK/descripción + contador "{N} activos · {M} desactivados".
  </Step>
  <Step title="Form expandible 'Nuevo {entidad}'" icon="plus" title-type="p">
    Click → form-grid con los campos (PK, descripción, etc.). Validaciones server-side: PK no vacío, formatos consistentes (uppercase IDs).
  </Step>
  <Step title="Edit inline por fila" icon="edit" title-type="p">
    Botón "Editar" reemplaza las celdas por inputs. Guardar / Cancelar. La PK NO es editable post-creación.
  </Step>
  <Step title="Desactivar / Reactivar" icon="x" title-type="p">
    Soft-delete con `activo=false`. Filas inactivas con opacity 0.5. Botón toggle. Confirm antes de desactivar.

    `desactivarAlmacenAction` valida que no haya stock con `disponible > 0` antes de aceptar (bloquea con mensaje específico "Este almacén tiene N SKUs con stock — hacé traspaso antes").
  </Step>
</Steps>

## SKUs (caso especial)

Catálogo más complejo. Cada fila:

- **AXAPTA** (PK) — clave interna WCG
- **SIATEL** — clave Telmex (opcional)
- **descripción** + **UM** (dropdown de `unidades_medida` activas)
- **precio** MXN
- **factor_alta_ftth** / **factor_mig_ftth** / **factor_mig_ftth_vieja** — multiplican el `altas` del wizard de OS para sugerir material

<Callout kind="info">
  Los SKUs históricos vienen de `Data/RECALCULO ALTA FO ACTUALIZADO.xlsx`. Importación inicial via `node scripts/build-telmex-import.mjs > scripts/telmex-import.sql`. Pero desde Día 0 ya **no es necesario** — el admin puede dar de alta SKUs uno por uno desde la UI.
</Callout>

## Permisos

| Acción | Roles |
|---|---|
| Ver catálogos | todos excepto **técnico** |
| Crear / editar / desactivar | **admin** únicamente |

RLS endurecida: cada tabla tiene policy `"Admin mutates X"` con
`current_user_role() = 'admin'` (con `WITH CHECK` también). SELECT abierto
a `authenticated using (true)` porque son data de referencia que todos
los flows necesitan.

## Server Actions

```typescript
// src/lib/actions/catalogos.ts

// Tecnicos (especifico)
crearTecnicoAction(input: TecnicoInput)
editarTecnicoAction(id, patch)
desactivarTecnicoAction(id)
reactivarTecnicoAction(id)

// Almacenes (especifico)
crearAlmacenAction(input: AlmacenInput)
editarAlmacenAction(id, patch)
desactivarAlmacenAction(id)  // bloquea si hay stock
reactivarAlmacenAction(id)

// SKUs (especifico con factores Telmex)
crearSKUAction(input: SKUInput)
editarSKUAction(sku, patch)
toggleActivoSKUAction(sku, activo)

// Genérico para los 5 simples (copes, empresas, tipos_tarea, unidades, clientes)
crearSimpleAction(table: SimpleTable, row)
editarSimpleAction(table, id, patch)
toggleActivoSimpleAction(table, id, activo)
```

Todas las mutations invalidan `_tecnicosCache` y `_almacenesCache` en `db.ts`
para que los próximos renders vean los datos frescos.

## Carga histórica (opcional)

Si querés sembrar los catálogos del Excel WCG original:

```bash
node scripts/build-seed.mjs > scripts/seed.sql
node scripts/build-telmex-import.mjs > scripts/telmex-import.sql
psql $SUPABASE_DB_URL -f scripts/telmex-import.sql
```

Pero la app **no depende de esto**. Un cliente nuevo arranca con DB vacía
y crea todo desde `/catalogos`. Ver [Día 0](/dia-cero) para el flow completo.
