## Qué muestra

`/almacen` es la vista del inventario en vivo. Cruza:

- **3 almacenes físicos** (ALM-VC, etc.) — stock central.
- **6 técnicos** — cada uno con su "mochila" (kit semanal asignado).

Cada fila es un par `(sku, ubicación)`. La tabla es densa: SKU, descripción, UM, disponible, reservado, total, último movimiento, min-stock, precio.

<Callout kind="info">
  El stock por técnico no es un almacén físico — es lo que se le **surtió** del kit semanal y aún no consume en sus OS. Al liquidar una OS, el material consumido se descuenta atómicamente de la mochila.
</Callout>

## Surtir a técnico (`/almacen/surtir`)

<Steps>
  <Step title="Elegir técnico y semana" icon="user" title-type="p">
    El almacenista selecciona destino. La semana es informativa (no es una FK).
  </Step>
  <Step title="Armar kit" icon="package" title-type="p">
    Línea por SKU con cantidad. El stock disponible se valida en vivo contra el almacén origen.
  </Step>
  <Step title="Confirmar" icon="check" title-type="p">
    `crearSalidaTecnicoAction` llama a la RPC `surtir_a_tecnico`: valida `disponible >= cantidad` (Bug 2 — sin esto el almacén iba negativo), decrementa origen via UPDATE y aumenta mochila via `INSERT…ON CONFLICT DO UPDATE`. Si el SKU no existe o no alcanza, lanza error específico (no es código `23514` — es un raise exception). No hay retry automático: el error sube al usuario.
  </Step>
</Steps>

## Reglas atómicas

<Callout kind="alert">
  Todas las operaciones de stock pasan por **upserts atómicos** en Supabase (`auditoría 822küg`). Esto evita el bug histórico de stock duplicado cuando dos almacenistas surtían al mismo tiempo.
</Callout>

```
src/lib/actions/almacen-movs.ts
  crearSalidaTecnicoAction(input)   // → RPC surtir_a_tecnico
  crearTraspasoAction(input)        // → RPC ejecutar_traspaso (origen ≠ destino)
  crearDevolucionAction(input)      // → RPC devolver_al_almacen (admin/almacenista/tecnico)
                                    // Todos via INSERT…ON CONFLICT DO UPDATE en inventory.
                                    // Helper privado `dedupePorSku` evita unique_violation
                                    // si la UI repite SKU (Bug 10).
```

## Tabla `inventory`

| Columna | Notas |
|---|---|
| `sku` | FK a `skus` |
| `almacen_id` | XOR con `tecnico_id` — un row es de un almacén **o** de una mochila |
| `tecnico_id` | XOR con `almacen_id` |
| `disponible` | Stock libre para uso |
| `reservado` | Stock comprometido a OS abiertas |
| `total` | Generado: `disponible + reservado` |
| `min_stock` | Para alertas de reorden |
| `ult_mov` | Timestamp del último update |
